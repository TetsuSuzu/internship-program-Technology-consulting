# 07. 内製化支援の文脈と、未経験者への課題の渡し方

## 課題
初対面・社会人未経験の学生に、抽象的な業務課題（内製化支援）をいきなり渡すと固まる。

## 鍵：内製化＝「自分でできるようになる」と翻訳する

DXCの支援の本質は「丸投げしない／手の内化」。学生語に訳すと：

> 内製化 ＝ 「人にやってもらう」から「自分でできるようになる」への変化

学生が毎日体験していること：
- 塾に全部やってもらう vs 自分で勉強できるようになる
- 答えを教わる vs 解き方が分かるようになる
- 「魚をもらう」 vs 「釣り方を覚える」

---

## 抽象的な課題を「渡さない」。体験させて、後から名前をつける

```
【誤】 業務課題を渡す → 解け（抽象的で動けない）
【正】 まず体験させる → 体験に名前をつける → 課題にする
```

### STEP1　「丸投げ」と「自分でできた」の差を体感させる
最初の小さな作業を2パターンで：
- ① 講師が全部やって見せる（学生は見てるだけ＝受け身）
- ② 学生が自分の手で同じことをやる（＝「できた！」）
→ 「どっちが身についた？嬉しい？」と問う。全員②と答える。

### STEP2　その感覚に名前をつける
> 「今の"自分でできた！"が、お客様が一番欲しいもの。
> ベンダーに丸投げじゃなく、自分の会社の人が自分でできるようになる——これを **内製化** と言う。
> それを手伝うのが、僕たちDXCの仕事です。」

（体験の"後"に言葉を置くから刺さる。`docs/01` の「種明かしは使わない」とは別物＝体験→言語化の王道）

### STEP3　ここで初めて課題を渡す（具体的に）
> あなたが今日できるようになったことを、
> **「次の人が、一人で同じことをできるようになる"学びのキット"」** にまとめてください。

| 難しい業務課題 | この課題 |
|--------------|---------|
| 経験が要る | 今日やったことが題材（経験不要） |
| 抽象的 | 「次の人ができるように」＝具体的ゴール |
| 内製化が分からない | 作る行為そのものが内製化 |

決定的な仕掛け：**自分のキットでペアの相手が本当にできるか試す**
> できたら成功。できなければ"丸投げ"になってる証拠 → 直す。
> ＝DXCの内製化支援の縮図。

---

## 初対面をどう乗り越えるか

| 順番 | やること | ねらい |
|------|---------|-------|
| ① 導入 | 30秒自己紹介＋「最近ハマってること」 | 緊張を解く |
| ② 軽い共同作業 | 2人で簡単タスク（例：リポジトリ名を相談して決める） | "一緒に決めた"を作る |
| ③ ペアで体験 | STEP1の作業をペアで | 共通体験で仲間化 |
| ④ 課題 | キット作り→相手で検証 | 「相手のために作る」で結束 |

→ 「相手が一人でできるよう助ける」課題自体が、初対面の壁とお題を同時に解く装置になる。
