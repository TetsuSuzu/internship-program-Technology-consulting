# 01. プログラムの大義名分

## 就業体験としての位置づけ

「就業体験」は **学生が企業の仕事の一部分を体験できること** が前提。
本プログラムのゴールは、**「DXCに入社してできる楽しい仕事」を体験すること**。

そのために、学生を「学ぶ人」ではなく **「お客様の課題を解くDXCの一員」** として扱う。

---

## テクノロジーコンサルタントの実務を分解する

参考：DXC Japan ブログ「企業変革のためのデジタル人材育成」
（内製化支援／デジタル人材育成の文脈）

| # | 普段の業務 | 本質 |
|---|-----------|------|
| ① | 現状理解・課題ヒアリング | お客様のビジネスを理解し、「ありたい姿」とのギャップを見つける |
| ② | 構想・企画策定 | DX/人材育成のロードマップ・プログラムを設計する |
| ③ | ワークショップ運営・伴走 | 問題解決WSで、お客様自身に手を動かさせる |
| ④ | クラウド技術の適用支援 | 日常業務の課題に、クラウドをどう使うか一緒に考え実装する |
| ⑤ | 振り返り・自分ごと化 | 「丸投げしない」「手の内化」できる状態へ導く |

→ 一言でいうと **「お客様が自分の力で変革できるように、技術と伴走で支援する仕事」**。

---

## 学生が体験できる部分

| 体験させる業務 | 学生が体験する形 |
|--------------|----------------|
| ③ 手を動かす | Codespaces / GitHub で実際に作る |
| ④ クラウド技術の適用 | 「身近な課題をITで解決」を疑似体験 |
| ⑤ 振り返り・自分ごと化 | 成果を発表し「自分でできた」実感を得る |

---

## キャッチコピー（案）

> **「あなたを"変革できる人"に育てる —— それが、DXCのテクノロジーコンサルタントの仕事です。」**

---

## 「種明かし」は使わない（重要な設計判断）

最後に「実はこれが仕事でした」と明かす演出は **弱い**。理由：
- 後出し＝就業体験になっていない（最中に「仕事をしている」自覚が持てない）
- 驚きが小さく「なるほど（で？）」で終わる

### 代わりに：最初から本物だと伝える

| | 旧（種明かし型） | 新（最初から本物型） |
|---|----------------|--------------------|
| 冒頭 | 普通に課題提示 | **「君は今日からDXC社員」と宣言** |
| 最中 | ただのハンズオン | **講師本人の現場エピソードを随所に** |
| 最後 | 「実は仕事でした」 | **先輩としての本音メッセージ** |
| 学生の体感 | 「へぇ」 | **「今、自分は本物の仕事をしている」** |

ラストは仕掛けの回収ではなく、講師（現役コンサルタント）の **素の言葉** で締める。
例：「お客様が"自分でできた"って笑う瞬間が、本当に楽しい。ピンと来たならDXCで待ってます。」
