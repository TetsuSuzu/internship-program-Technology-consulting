# 🧪 Playwright テスト手順書（発展・任意）

> **このモジュールのゴール**
> 自分が作ったページが「ちゃんと表示されているか」を、**自動テスト**で確認できるようになる！
> （緑になれば合格 ✅ ／ 赤なら直すところがある ❌）

対象：早く終わった人・もう少し挑戦したい人向けの **発展課題**
所要時間：約 20〜30 分 ／ 場所：Codespaces のターミナル

---

## 💡 そもそも「自動テスト」って？

人間が毎回目で確認する代わりに、**機械が自動でページをチェック**してくれる仕組みです。

> DXCのエンジニアは「作る」だけでなく「**ちゃんと動くか**」を自動でテストします。
> 自分で品質を保証できる ＝ 人任せにしない ＝ **内製化** の考え方そのものです。

---

## STEP 1　準備（最初の1回だけ）

Codespaces のターミナルで、次のコマンドを順番に実行します。

```bash
# 1. Playwright をインストール
npm install -D @playwright/test

# 2. テスト用ブラウザを入れる（少し時間がかかります）
npx playwright install --with-deps chromium
```

> ⏳ 2番目のコマンドはダウンロードに数分かかることがあります。気長に待ってOK。

---

## STEP 2　テストを実行する

```bash
npx playwright test
```

成功すると、こんな表示になります 👇

```
Running 5 tests using 1 worker
  5 passed (3s)
```

→ **「passed（合格）」が出れば、あなたのページは正しく表示されています！** 🎉

---

## STEP 3　結果をレポートで見る

```bash
npx playwright show-report
```

- ポートが転送され、右下に「Open in Browser」通知が出る → クリック
- 各テストの合否が一覧で見られます（赤い項目があれば、そこが直すところ）

> 📸 **【スクショ：テストレポート画面】**

---

## STEP 4（やってみよう）わざと「赤」にしてみる 🔴

テストが何をしているか体感するための実験です。

1. `handson/index.html` の `<h1>` の**見出しを丸ごと消して**保存する
2. もう一度 `npx playwright test` を実行する
3. → **「見出し（h1）が表示されている」テストが赤くなる**（失敗する）

これがテストの役目です。
**「壊れたことを、人より先に機械が教えてくれる」** ——だから安心して直せる。
確認できたら、消した見出しを元に戻しておきましょう。

---

## 🆘 こまったときは

| 症状 | 対処 |
|------|------|
| `python3` が無いと出る | `playwright.config.js` の `python3` を `python` に変える |
| ブラウザのインストールで失敗 | `npx playwright install chromium`（`--with-deps` を外す）で再挑戦 |
| `port 8000 が使用中` | 既に別のサーバが起動中。古いターミナルを閉じる |
| ぜんぶ分からない | 画面を見せて講師・メンバーに聞く！ |
